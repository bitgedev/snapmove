# Snapmove PRD

---

## 1. 문제 정의

운동을 꾸준히 하는 사람도 자신의 기록을 체계적으로 관리하고 공유할 수 있는 도구가 부족하다. Snapmove는 루틴 관리부터 인증샷 카드 생성까지 운동 기록의 전 과정을 하나의 앱에서 해결한다.

---

## 2. 타겟 사용자

운동 루틴을 관리하고 싶지만 기록이 흩어져 있고, SNS에 운동 인증을 올리고 싶지만 직접 편집하기 번거로운 20~30대 운동 입문자 및 중급자.

---

## 3. 제안 해결책

루틴 생성부터 운동 기록, 감성적인 인증샷 카드 자동 생성 및 저장까지 하나의 앱에서 처리할 수 있는 올인원 운동 트래커를 제공한다. 실시간 타이머 대신 **수동 입력 방식**을 채택해 웹 백그라운드 제한을 회피하고 회고 중심의 습관 형성에 집중한다.

---

## 4. 목표 및 성공 지표

운동 기록과 공유 경험을 단순화하여 사용자가 꾸준히 앱을 사용하게 만드는 것을 목표로 한다. 성공 지표는 루틴 완료율, 인증샷 카드 저장/공유 횟수, 재방문율로 측정한다.

---

## 5. 핵심 기능 (MVP)

**인증 (Auth)** ✅ 실제 구현

- Supabase Auth 이메일 회원가입 / 로그인
- Next.js middleware 라우팅 보호 (미인증 시 `/login` redirect)
- 로그인 후 첫 화면은 `/dashboard`

**대시보드** ⚠️ mock 데이터

- 이번 주 운동 달성 현황 — 주간 게이지 바 (목표 횟수 대비 완료 횟수, 요일별 dot 마커)
- 최근 운동 3건 요약 카드
- 전체 기록 보기 링크 → `/history`

**루틴 관리** ⚠️ mock 데이터

- 루틴 생성 / 수정 / 삭제 (모바일 최적화 Drawer UI) — UI만, 저장 없음
- 루틴 내 운동 추가 / 삭제
- 운동 카테고리 분류: **근력 / 유산소 / 유연성 / 기타** (근력 선택 시 세부 근육군 추가 선택)
- 운동별 세트 / 횟수 / 무게 / 휴식 시간 설정

**운동 기록 (수동 입력 방식)** ✅ 실제 구현

- 루틴 실행 후 세트별 실제 수행 기록 (무게·횟수 직접 입력)
- 실시간 타이머 없음 — 완료 시 운동 시간 수동 입력
- 시간 퀵 버튼: 15분 / 30분 / 60분 / 90분 + 직접 입력 필드
- 운동 완료 시 Supabase DB에 세션 저장

**인증샷 카드** ✅ 실제 구현

- 운동 완료 시 `canvas-confetti` 폭죽 효과 + 카드 생성 유도
- 사진 촬영 / 갤러리 업로드 (`<input type="file" capture="environment">`) — 모바일 브라우저에서 카메라 직접 연동
- 사진 + 운동 데이터(날짜·운동 목록·총 볼륨·운동 시간)를 합성한 감성 카드
- 인스타그램 비율 대응: 1:1 / 4:5 전환 토글
- html2canvas로 인증샷 카드 PNG 저장
- 이미지 저장 / 공유 버튼

**운동 히스토리 (캘린더 뷰)** ⚠️ mock 데이터

- 월간 캘린더 그리드 — 운동한 날 teal dot 표시
- 날짜 선택 시 해당 날 운동 카드 목록 표시
- 과거 운동의 인증샷 카드 재확인

## 6. 기술 스택

| 구분 | 기술 |
|---|---|
| Frontend | Next.js 14+ App Router, TypeScript, Tailwind CSS v4, Shadcn UI |
| 상태 관리 | React 상태 (Client Component) |
| UI 패턴 | Server Component 우선, 입력 폼은 Shadcn Drawer (바텀 시트) |
| 인증 / DB | Supabase Auth + Supabase Database (부분 연동) |
| 인증샷 카드 저장 | html2canvas |
| 폭죽 효과 | canvas-confetti |
| 배포 | Vercel |

---

## 7. 페이지 구조

| 페이지 | 경로 | 설명 |
|---|---|---|
| 랜딩 | `/` | 서비스 소개 및 로그인 유도 |
| 로그인 | `/login` | 이메일 로그인 → `/dashboard` 이동 |
| 회원가입 | `/signup` | 이메일 회원가입 → `/dashboard` 이동 |
| 대시보드 | `/dashboard` | 주간 게이지, 최근 운동 요약 — **로그인 후 첫 화면** |
| 루틴 목록 | `/routines` | 내 루틴 카드 그리드 |
| 루틴 상세 | `/routines/[id]` | 운동 목록, 운동 시작 버튼 |
| 운동 실행 | `/workout/[id]` | 세트별 수동 입력, 완료 시 사진 업로드 + 인증샷 카드 생성 |
| 히스토리 | `/history` | 달력 뷰 + 선택 날짜 운동 기록 |

---

## 8. UX 원칙

- **모바일 퍼스트**: 375px 기준 설계, 입력 폼은 Drawer(바텀 시트)로 화면 전체 활용
- **수동 입력 중심**: 실시간 추적 대신 운동 후 회고 입력으로 부담 최소화
- **성취감 부여**: 운동 완료 시 폭죽 효과 → 카드 생성 유도 → SNS 공유까지 자연스러운 플로우
- **단일 컬러 시스템**: 연한 민트(teal) 단일 계열, 보조 강조색 없음
